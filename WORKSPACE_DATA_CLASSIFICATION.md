# Workspace Data Classification

Дата: 2026-02-26 (UTC)
Цель: разделить данные `~/.openclaw/workspace` на **ценные** (в коммит текущего workspace-репо) и **неценные/служебные** (не коммитить в этот репозиторий).

## 1) Ценные данные (коммитим)

### Личность/контекст ассистента (критично)
- `AGENTS.md`
- `SOUL.md`
- `USER.md`
- `IDENTITY.md`
- `TOOLS.md`
- `HEARTBEAT.md`

### Память и знания (критично)
- `memory/` (включая `memory/*.md`, тематические `.md/.json`)

### Операционные документы
- `docs/` (инструкции и локальные runbook'и)

## 2) Ценные, но **не для этого репо** (хранить отдельно)

Это самостоятельные проекты с собственным git-жизненным циклом:
- `openclaw/` (nested git repo)
- `ouroboros/` (nested git repo)

Решение: поддерживать и бэкапить их через их собственные upstream-репозитории, а не тащить в текущий workspace-репо.

## 3) Неценные/генерируемые/тяжёлые (не коммитить)

### Временные/служебные
- `.openclaw/`
- `.pi/`
- `.trash/`
- `tmp/`
- `out/`

### Медиа/артефакты/кэш
- `downloads/`
- `researches/` (в особенности `media/`, большие выгрузки)
- `comic_output/`
- `drawn_comic/`
- `video2027.mp4`
- `*.png`, `*.jpg` сравнительные/промежуточные файлы в корне
- `tg2027.html` (если это экспорт/артефакт, а не исходник знания)

### Причина исключения
- большой размер,
- легко регенерируется,
- не влияет на восстановление личности/памяти ассистента,
- создаёт шум в истории.

## 4) Коммит-политика для workspace

Коммитить в этот репозиторий только:
- файлы личности/правил,
- память,
- docs.

Не коммитить:
- бинарные артефакты, выгрузки, медиа, временные каталоги,
- nested репозитории (`openclaw/`, `ouroboros/`) целиком.

## 5) Что включено в текущий коммит

- `WORKSPACE_DATA_CLASSIFICATION.md` (этот файл)
- `AGENTS.md`
- `SOUL.md`
- `USER.md`
- `IDENTITY.md`
- `TOOLS.md`
- `HEARTBEAT.md`
- `memory/`
- `docs/`
