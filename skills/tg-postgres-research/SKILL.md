# TG PixelNAS Research (универсальный)

1. Прочитать `/home/adam/dev/tools/q-tools/docs/tg-researches.md`.
2. Если в сообщении есть `t.me`-ссылки без явного ТЗ:
   - извлечь из ссылок username/источники,
   - определить режим: целиком источник(и) или точечные посты.
3. Базовый режим (по умолчанию): **text-only**.
   - синк в PostgreSQL через:
     - `cd ~/dev/tools/q-tools`
     - `./bin/q tg-text-sync-pg --sources <comma-separated>`
   - медиа не скачивать и не сохранять.
4. Если пользователь явно просит медиа:
   - запускать отдельный media-aware сбор,
   - в отчёте явно помечать, что медиа-режим был включён.
5. Для точечных сообщений:
   - брать конкретные `channel/msg_id` из ссылок,
   - сохранять в те же таблицы PostgreSQL (только выбранные записи, без полного прохода).
6. После синка выполнять тематический анализ по запросу и формировать строго:
   - один `.md` файл,
   - один красивый `.html` файл для отправки.
7. Финальный ответ пользователю должен содержать:
   - какие источники/ссылки обработаны,
   - сколько текстовых сообщений сохранено/обновлено,
   - где данные в PostgreSQL (`tg_research.messages`, `tg_research.sync_state`),
   - путь к `.md` и `.html`.
