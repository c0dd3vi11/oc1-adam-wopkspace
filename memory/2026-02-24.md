
## 2026-02-24 — LostFilm automation + emoji pipeline overhaul (durable)

- LostFilm pipeline was redesigned to separate rendering and publishing for emoji assets:
  - `q lostfilm-emoji-render` generates local source PNG assets into `data/tg-emoji/lostfilm-dot-tv-pack/`.
  - `q lostfilm-emoji-apply` applies those rendered assets to Telegram custom emoji sticker set and updates emoji ids in map.
  - `q lostfilm-emoji-sync` remains as legacy wrapper (render + apply).
- Added `data/tg-emoji/lostfilm-dot-tv-pack/AGENTS.md` documenting mandatory screenshot-diff review workflow before applying pack changes.
- Preview workflow established:
  - local HTTP server on `127.0.0.1:19345` used for reviewing `preview.html`.
  - repeated screenshot-diff iterations are now standard for shape/curvature tweaks.
- Introduced local source cache for poster/logo inputs:
  - cache path: `caches/lostfilm/`.
  - files now saved with inferred extensions (jpg/png/ico/etc), not opaque `.img` only.
- LostFilm bot routing changed to strict dedicated token:
  - only `TELEGRAM_BOT_TOKEN_LOSTFILMDOTTVBOT` is used for LostFilm flows.
  - fallback to generic `TELEGRAM_BOT_TOKEN` removed.
- Telegram custom emoji pack migrated to new bot set:
  - active pack: `https://t.me/addstickers/LostFilmDotTV_by_LostFilmDotTVBot`.
  - all scripts switched to this pack name.
- Publishing logic changed:
  - channel `@LostFilmDotTV` now publishes full RSS stream (no series-only filter; includes movies/items as-is).
  - favorites are sent separately to Anton's DM via `@LostFilmDotTVBot` as a single list, only on new favorites.
  - `lostfilm-filtered.json` kept as technical report feed to test channel.
- Fetch payload now includes precise UTC time (no seconds):
  - `date` format: `DD.MM.YYYY HH:MM UTC`.
- Fixed emoji mapping issue causing repeated TV fallback icon:
  - mapping now uses per-item key (`item_key`) rather than only `series_slug`.
  - supports series and movies (`series:*`, `movie:*`, fallback hash key).
- Added orchestrator to reduce cron timeout risk:
  - `scripts/lostfilm-run-cycle.py` runs staged cycle: fetch -> emoji-render -> emoji-apply -> publish.
  - emoji apply treated non-fatal to avoid whole cycle failure on sticker API slowness/flood.
- Telegram Bot API docs sync flow expanded:
  - sync now maintains raw/html, markdown, structured json and semantic-diff summary in test notifications.
  - daily docs cron remains active, plus BotNews watcher cron triggers out-of-band sync on new posts.
