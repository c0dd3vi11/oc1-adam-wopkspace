## Cursor Ultra integration idea (Anton)

- Discussed using Cursor Ultra subscription from OpenClaw without patching OpenClaw core provider logic.
- Proposed architecture: run a local shim service on `127.0.0.1` with OpenAI-compatible `/v1` API, and configure OpenClaw Custom Provider to target that URL.
- Shim would call `cursor-agent` in non-interactive mode (`--print`, JSON output), map responses back to OpenAI-compatible format.
- Key safety constraint: keep Cursor in "brain-only" role (ask/plan style), while all real actions (shell/tools/credentials) stay in OpenClaw.
- Security notes:
  - Do not pass OpenClaw secrets/credentials env vars into Cursor process.
  - Run Cursor with constrained environment/workdir.
  - Avoid allowing Cursor to execute autonomous shell/write actions.
- Next step planned: draft secure launch template + minimal shim contract (request/response/errors/timeouts).
